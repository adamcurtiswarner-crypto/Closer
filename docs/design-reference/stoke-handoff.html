01
What is Stoke?
A relationship operating system built around four daily engines.
1
Engine 1
Learn
One morning question. Five seconds. The partner never sees the raw answer — they receive a translated forecast instead. Privacy creates honesty.
Morning check-in
2
Engine 2
Action
The tennis match. Send one word. Partner guesses why. Reveal. AI turns the exchange into a conversation naturally. Zero therapy.
Today's Spark
3
Engine 3
Reflect
An evening micro-scorecard. Heart rating + one chip selection. Stoke learns what actually increases connection over time.
Evening Reflection
4
Engine 4
Coach
AI surface personal pattern insights. Not generic advice. "When you share dinner twice a week your score rises 34%." Warm, never preachy.
Insights tab
5
Surprise
Mission
Random, unexpected nudges. Not daily. A full-screen moment that asks for one small act. No explanation why. Just the ask.
Push notification
6
Bonus
Curiosity Loop
Guess Jordan's preference. Coffee in Paris or beach in Greece? Quick binary choice game that builds partner knowledge score over time.
Today tab · sub-screen

06
Tone-on-Tone Shapes
Organic, whimsical background texture. Referenced directly from the audio player screen.
Coral bg — ellipses + band
Large ellipse top-right, rotated band mid, small ellipse bottom-left. All rgba(255,255,255,.06–.10)
Black bg — subtle ellipses
Same pattern but opacity much lower (.04–.05). Barely visible — just enough texture.
Purple bg — accent only
Just two ellipses. Used only on Stoke Noticed insight cards — never full screens.

Key rule: Shapes are always ellipses and/or one gently rotated wide rectangle band. Never spokes. Never geometric lines. Never hard angles. The shapes should feel like they grew there — not like a pattern was applied. Keep opacity between .04 and .12 maximum.
07
Screen Architecture
The app cycles through four engines daily. This is the navigation map.
Navigation structure
// Tab bar (5 tabs)
Home         → Good Morning greeting + Daily Thought card + Recommended cards
Today        → Engine 2 spark + partner guess game + curiosity loop
Memories     → Past sparks, shared moments, connection history
Insights     → Engine 4 coach — personal pattern data
Settings     → Profile, partner, notification prefs

// Modal / full-screen overlays
MorningCheckin    → Push at 8am. One question. Dismiss after answering.
EveningReflection → Push at 9pm. Heart score + chips. ~30 seconds.
SurpriseMission   → Random push. Full coral screen. "I'm in" or skip.
AudioPlayer       → Full coral screen. Existing design. Keep exactly.
Daily user journey
8:00am   Push notification → MorningCheckin modal (Engine 1)
           ↓ answer → dismissed → partner receives forecast
Anytime  Home tab → tap Today's Spark card
           ↓ TodaySpark screen — pick/type a word, send
           ↓ partner gets notification → opens PartnerGuess screen
           ↓ submits guess → both reveal → AI sends conversation nudge
Random   Push → SurpriseMission full-screen (Engine 2 bonus)
9:00pm   Push → EveningReflection modal (Engine 3)
           ↓ heart score + chips → Stoke calculates insight
Insights Tab → CoachInsights screen (Engine 4)
           ↓ personal stats, highlighted recommendation, momentum bars

10
Dev Setup Checklist
Five steps from zero to running.
1
Add the font to index.html
Paste the Google Fonts <link> tag from Section 03 into the <head> of your index.html. Weights: 600, 700, 800, 900 + italic 700.
2
Copy the design tokens
Create src/constants/theme.js and paste the Colors, Typography, Radius, and Spacing objects from Section 08. Import from here everywhere — no hardcoded hex values in components.
3
Place the screens file
Copy StokeScreens.jsx into src/screens/. Each screen is a named export: TodaySpark, PartnerGuess, MorningCheckin, SurpriseMission, EveningReflection, CoachInsights.
4
Wire into your navigator
Using React Navigation? Register each screen in your stack. The SurpriseMission screen is designed as a modal — set presentation: 'modal' in your stack config.
5
Questions?
All design decisions in this document have a reason behind them. If something isn't clear — spacing, a color choice, when to use purple vs coral — refer back to Sections 02–06 first, then reach out.
